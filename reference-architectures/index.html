Reference Architectures

Four evergreen architectures for the four disciplines.

Each architecture is a working reference for one slice of The 4‑Discipline Stack — not a vendor pitch, not a marketecture diagram. Each shows the layers, the standards that govern them, the named tools (commercial + OSS) you’d pick today, and the cross-cutting substrate they share.

Applied GenAI · reference architecture

Regulated GenAI Platform.

Model gateway, RAG, guardrails, evals, audit-evidence pipeline, observability. Aligned to NIST AI RMF + EU AI Act + ISO/IEC 42001. The platform you build before squads build their own.

View architecture → Platform Eng · DevSecOps

DevSecOps SLSA L3+ Paved Path.

Trunk-based CI/CD with signed provenance, workload identity (OIDC), SBOM-to-owner alerting, policy-as-code admission control. Aligned to NIST SSDF + SLSA + EO 14028.

View architecture → Data Modernisation

Modern Data Platform.

Lakehouse + governed mesh: ingestion, contracts, lineage, catalogue, semantic layer, ML/AI feature store, audit retention. Aligned to DCAM / DAMA-DMBOK / ISO 8000 / regulator data-residency.

View architecture → Platform Engineering

Platform Engineering IDP.

Internal developer platform with paved paths, golden templates, service catalogue, scorecards, self-service. Aligned to CNCF Platform Eng Maturity + Team Topologies + DORA.
